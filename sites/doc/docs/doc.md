# Документация по модулю управления документами

Этот документ описывает архитектуру, структуру базы данных и логику работы модуля управления документами в рамках проекта.

## 1. Общая концепция

Модуль предназначен для создания иерархической структуры каталогов и управления файлами внутри них. Он позволяет загружать, редактировать, скачивать, версионировать и удалять документы.

Основной интерфейс реализован с помощью JavaScript-библиотеки **jsTree**, что обеспечивает интерактивное взаимодействие с деревом каталогов (перетаскивание, создание, удаление) без перезагрузки страницы.

## 2. Архитектура и ключевые файлы

Модуль реализован в соответствии с MVC-подобной архитектурой фреймворка.

*   **`sahmatka/fw/controllers/ctr__doc.php`**: Отвечает за **структуру** каталогов и за **содержимое** (файлы). Он управляет логикой загрузки и редактирования файлов, обработкой скачивания и генерацией ZIP-архивов. **Представление** (HTML, CSS, JS) вынесено в отдельные файлы шаблонов, что отделяет логику от отображения.

## 3. Структура базы данных

В основе модуля лежат две основные таблицы из `m2profi_doc.sql`:

### Таблица `dir`
Хранит информацию о каталогах (папках).

| Поле            | Тип         | Описание                                                               |
| --------------- | ----------- | ---------------------------------------------------------------------- |
| `dir_id`        | `int`       | **Primary Key**, уникальный идентификатор каталога.                    |
| `dir_title`     | `varchar`   | Название каталога, которое видит пользователь.                         |
| `parent_dir_id` | `int`       | ID родительского каталога. `0` для корневых каталогов.                 |
| `dir_name`      | `varchar`   | Системное имя (транслит), используется для построения `dir_path`.      |
| `dir_level`     | `int`       | Уровень вложенности каталога.                                          |
| `dir_path`      | `varchar`   | Полный системный путь до каталога (например, `root/subfolder`).        |
| `order`         | `int`       | Порядок сортировки каталогов на одном уровне.                          |
| `del`           | `int`       | Флаг "мягкого" удаления. `1` означает, что каталог удален.             |

### Таблица `files2node`
Ключевая таблица, хранящая информацию о файлах и их связях с каталогами.

| Поле            | Тип         | Описание                                                                                             |
| --------------- | ----------- | ---------------------------------------------------------------------------------------------------- |
| `files2node_id` | `int`       | **Primary Key**, уникальный идентификатор файла.                                                      |
| `node_type`     | `varchar`   | **Ключевое поле связи.** Для документов всегда имеет значение **`'doc_dir'`**.                        |
| `node_id`       | `int`       | **Внешний ключ.** ID каталога из таблицы `dir`, к которому привязан файл.                             |
| `name`          | `varchar`   | Имя файла, которое было у него при загрузке.                                                         |
| `caption`       | `varchar`   | Заголовок файла, который видит пользователь. Если пустой, используется `name`.                       |
| `docdate`       | `date`      | Дата документа (указывается пользователем).                                                          |
| `comment`       | `text`      | Комментарий к файлу.                                                                                 |
| `link`          | `varchar`   | Полная публичная ссылка на файл.                                                                      |
| `puth`          | `varchar`   | Относительный путь к файлу на сервере.                                                               |
| `uptime`        | `int`       | Timestamp последнего обновления записи.                                                              |
| `del`           | `int`       | Флаг "мягкого" удаления. `1` означает, что файл удален.                                              |

## 4. Логика работы

### 4.1. Дерево документов (`jsTree`)

Дерево документов построено на библиотеке **`jsTree`**. Оно обеспечивает весь интерактивный функционал для управления папками и файлами.

*   **Источник данных**: `jsTree` получает данные асинхронно от метода `act__get_tree_data` контроллера `ctr__doc`. Метод возвращает JSON-структуру, описывающую папки и файлы.

*   **Основные плагины и их функции**:
    *   **`dnd` (Drag-and-Drop)**: Реализован функционал для перемещения файлов и папок. При завершении перетаскивания `jsTree` отправляет AJAX-запрос методу `act__move_node`, который обновляет `parent_dir_id` для папок или `node_id` для файлов в базе данных.
    *   **`state`**: Состояние дерева (информация об открытых и закрытых ветках) сохраняется в `localStorage` браузера. Это позволяет восстанавливать вид дерева при перезагрузке страницы.
    *   **`search`**: Поиск по документам работает через AJAX. При вводе текста в поле поиска `jsTree` отправляет запрос методу `act__search_tree`. Бэкенд ищет совпадения в названиях и заголовках файлов и возвращает ID найденных узлов. `jsTree` автоматически скрывает все узлы, кроме найденных и их родительских папок.
    *   **`types`**: Используются разные иконки для папок и файлов, что улучшает визуальное восприятие структуры.

*   **Кастомизация**: Внешний вид дерева полностью стилизован под дизайн проекта. Сохранен и интегрирован весь кастомный функционал, включая кнопки для добавления документов в шапке каталога и ссылки, открывающие карточки документов.

### 4.2. Управление файлами (`ctr__doc.php`)

*   **Добавление/Редактирование**:
    1.  Пользователь нажимает "+" в заголовке каталога.
    2.  Открывается форма, которую генерирует метод **`act__edit()`**, используя шаблон `edit_form.php`.
    3.  После заполнения формы и загрузки файла данные отправляются `POST`-запросом на тот же `act__edit()`.
    4.  Метод сохраняет/обновляет запись в `files2node`, устанавливая `node_type = 'doc_dir'` и `node_id` равным ID текущего каталога.
*   **Скачивание**:
    1.  Пользователь кликает по файлу в дереве, открывается карточка файла, которую генерирует **`act__card()`** с помощью шаблона `document_card.php`.
    2.  В карточке есть ссылка на скачивание, ведущая на **`act__download()`** с ID файла.
    3.  Метод находит запись в `files2node`, получает путь к файлу (`puth`) и отдает его браузеру.
*   **Скачивание архива**: Метод **`act__zipdir()`** собирает все файлы из каталога, создает ZIP-архив и отдает его на скачивание.

### 4.3. Шаблоны и Представление

Вся логика представления вынесена в отдельные файлы шаблонов для четкого разделения логики и отображения.

Шаблоны находятся в директории `sahmatka/fw/templates/doc/`:

*   **`edit_form.php`**: Форма добавления и редактирования документа (`act__edit()`).
*   **`document_card.php`**: Карточка с детальной информацией о файле (`act__card()`).
*   **`document_tree.php`**: HTML-разметка, стили и JavaScript-код для инициализации и работы `jsTree` (`show_tree()`).
