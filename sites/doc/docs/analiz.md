# Детальный анализ: Жизненный цикл запроса

Этот документ описывает пошаговый процесс обработки HTTP-запроса фреймворком, от получения запроса до отправки ответа пользователю.

## Шаг 1: Точка входа

Пользовательский запрос всегда попадает в одну из трех основных точек входа:

1.  **`sahmatka/ctrind.php`**:
    - **Назначение**: Основная точка входа для стандартных страниц, которые требуют полного HTML-шаблона (шапка, контент, подвал).
    - **Особенность**: Оборачивает результат работы контроллера в полноценную HTML-структуру.

2.  **`sahmatka/ajax_router.php`**:
    - **Назначение**: Для обработки AJAX-запросов.
    - **Особенность**: Выводит "сырой" результат работы контроллера, без какой-либо HTML-обвязки. Устанавливает специфичные для AJAX HTTP-заголовки (например, `no-cache`). **Важно:** текущая проверка авторизации `if( $_SESSION['sh_login'] || 1==1)` фактически отключена, что является уязвимостью.

3.  **`sahmatka/iframe_router.php`**:
    - **Назначение**: Для страниц, отображаемых внутри `<iframe>`.
    - **Особенность**: Оборачивает результат работы контроллера в минимальную HTML-структуру (содержит `<head>` с подключением CSS/JS и `<body>`). Как и `ajax_router`, имеет отключенную проверку авторизации.

## Шаг 2: Инициализация и конфигурация (`sahmatka/config.php`)

Каждая точка входа первым делом подключает `sahmatka/config.php`. Этот файл выполняет ключевые действия по инициализации:

1.  **Настройка окружения**: Включает или отключает отображение ошибок в зависимости от наличия `$_GET['dev']`.
2.  **Запуск сессии**: `session_start()` для работы с сессионными переменными.
3.  **Подключение зависимостей**: Подключает все основные файлы ядра из директории `/core/`, включая:
    - `functions.php`
    - `classes.php`
    - `classes/controller.php` (базовый класс `ctr__`)
    - `classes/router.php` (класс `router`)
4.  **Соединение с БД**: Устанавливает соединение с базой данных MySQL.
5.  **Создание экземпляра маршрутизатора**: Создает глобально доступный объект маршрутизатора: `$r = new router();`. Этот объект будет использоваться для обработки запроса.

## Шаг 3: Аутентификация ("Страж" в `sahmatka/incudes_/header.php`)

Для `ctrind.php` следующим шагом является подключение `sahmatka/incudes_/header.php`. Этот файл действует как единый центр аутентификации:

1.  **Проверка выхода**: Если в URL есть параметр `exit`, сессия пользователя очищается.
2.  **Проверка сессии**: Если сессия (`$_SESSION['sh_login']`) существует, данные пользователя обновляются из БД. Если пользователь в БД не найден, сессия принудительно очищается.
3.  **Обработка формы входа**: Если был отправлен POST-запрос с данными формы входа, `header.php` **самостоятельно** обрабатывает его:
    - Проверяет данные пользователя по базе.
    - В случае успеха устанавливает сессию.
    - **Немедленно выполняет редирект** на страницу по умолчанию (`/sahmatka/ctrind.php?ctr=doc&act=index`).
    - **Завершает выполнение скрипта** (`exit()`), чтобы предотвратить дальнейшую обработку.
4.  **Блокировка**: Если по итогам всех проверок сессия отсутствует, `header.php` отображает HTML-форму входа и также **завершает выполнение скрипта**.

Таким образом, дальнейшие шаги (маршрутизация и выполнение контроллера) происходят только в том случае, если у пользователя уже есть действительная сессия.

## Шаг 4: Маршрутизация

Если аутентификация прошла успешно, управление передается механизму маршрутизации.

1.  **Исполнитель (`sahmatka/router.php`)**: Точки входа (`ctrind.php`, `ajax_router.php` и др.) подключают этот небольшой скрипт. Его единственная задача — вызвать основной метод обработки у глобального объекта `$r`: `$r->action_content()`.

2.  **Ядро (`core/classes/router.php`)**:
    - Метод `action_content()` определяет, какой контроллер и действие (метод) нужно вызвать, на основе GET-параметров `ctr` и `act`. По умолчанию используются `index` и `index`.
    - Он формирует имя класса контроллера (например, `ctr__doc`) и имя метода (`act__index`).
    - С помощью метода `get_object()` он находит файл контроллера в директории `sahmatka/fw/controllers/`, подключает его и создает экземпляр класса.
    - **Важно**: В классе есть метод `check_user_rules()`, который должен проверять права доступа, но на данный момент он всегда возвращает `true`, то есть проверка прав отключена.
    - В конце, он вызывает нужный метод у объекта контроллера (например, `$obj->act__index()`).

## Шаг 5: Выполнение контроллера

1.  **Базовый контроллер (`core/classes/controller.php`)**: Все контроллеры в `sahmatka/fw/controllers/` наследуют класс `ctr__`. Этот базовый класс предоставляет общую функциональность, в первую очередь — метод `tpl()`.

2.  **Метод `tpl()`**: Это простой, но эффективный шаблонизатор. Он принимает массив данных, имя контроллера (для определения папки с шаблонами) и имя шаблона. С помощью буферизации вывода (`ob_start`, `include`, `ob_get_clean`) он выполняет PHP-файл шаблона, передавая ему данные, и возвращает результат в виде HTML-строки.

3.  **Конкретный контроллер** (например, `ctr__doc.php`):
    - Его метод (например, `act__index()`) выполняет основную бизнес-логику: получает данные из модели (напрямую из БД в данном случае), обрабатывает их.
    - В конце своей работы он вызывает `$this->tpl(...)`, чтобы сгенерировать HTML-представление на основе шаблона из `sahmatka/fw/templates/doc/` и передает его на вывод.

## Шаг 6: Завершение ответа

1.  **Для `ctrind.php`**: Результат работы контроллера (HTML-код) захватывается в переменную с помощью буферизации вывода. Затем этот HTML вставляется в основной шаблон сайта между `header.php` и `foother.php`.

2.  **Для `ajax_router.php`**: Результат работы контроллера выводится напрямую, без изменений.

3.  **Для `iframe_router.php`**: Результат вставляется внутрь минимальной HTML-страницы.
