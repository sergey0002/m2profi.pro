# Архитектура фреймворка: Подробное руководство

Этот документ представляет собой исчерпывающее руководство по архитектуре фреймворка. Он предназначен для разработчиков, желающих понять ключевые принципы его работы, от жизненного цикла запроса до создания собственных компонентов.

## Часть 1: Основы и структура проекта

### 1.1. Ключевые директории

Правильная организация файлов — залог понятного и поддерживаемого проекта. В нашем фреймворке принята следующая структура:

- **/core/**: "Сердце" фреймворка. Здесь лежат базовые классы, которые используются всеми сайтами, работающими на этой системе.
    - **/core/classes/**: Содержит самые важные классы, такие как `router.php` (маршрутизатор) и `controller.php` (базовый контроллер).

- **/sahmatka/fw/**: Ядро конкретного сайта ("Шахматка").
    - **/sahmatka/fw/controllers/**: Директория для классов-контроллеров. **Правило именования:** `ctr__{имя_контроллера}.php`. Например, `ctr__doc.php`.
    - **/sahmatka/fw/templates/**: Файлы представлений (HTML-шаблоны). Они сгруппированы в папки, названные в честь контроллеров, которые их используют. Например, шаблоны для контроллера `doc` лежат в `/sahmatka/fw/templates/doc/`.

- **/sahmatka/incudes_/**: Общие для всех страниц элементы, такие как шапка (`header.php`) и подвал (`foother.php`).

### 1.2. Конфигурация и точка входа

Любой запрос к сайту начинается с одного из файлов-точек входа. Главная из них — `sahmatka/ctrind.php`.

Первое, что делает любая точка входа, — это подключение конфигурационного файла `sahmatka/config.php`. Этот файл выполняет критически важные задачи:
1.  **Настройка окружения**: Включает или отключает отображение ошибок PHP.
2.  **Соединение с базой данных**: Устанавливает подключение к MySQL.
3.  **Конфигурация доменов**: Определяет доменные имена для разных частей проекта.
4.  **Подключение ядра**: Загружает все базовые классы из `/core/classes/`.
5.  **Создание маршрутизатора**: Создает глобальный объект `$r = new router();`, который будет управлять всем процессом обработки запроса.

### 1.3. Конфигурация доменов

Для удобства управления и гибкости развертывания все доменные имена и пути к файлам вынесены в конфигурационный файл `sahmatka/config.php`.

**Пример конфигурации:**
```php
$GLOBALS['config']['domains']['doc'] = 'https://doc.m2profi.pro';
$GLOBALS['config']['domains']['main'] = 'https://m2profi.pro';
$GLOBALS['config']['domains']['em'] = 'https://em.m2profi.pro';
$GLOBALS['config']['keys_base_dir'] = '/home/m2profi/web/m2profi.pro/public_html/sites/em/keysbase';
$GLOBALS['config']['keys_base_url'] = 'https://em.m2profi.pro/keysbase/';
```

**Правила использования:**
-   **Никогда не используйте жестко закодированные домены или пути в коде.**
-   Для доступа к доменам используйте `$GLOBALS['config']['domains']['имя_домена']`.
-   Для доступа к путям используйте `$GLOBALS['config']['имя_пути']`.

Это позволяет легко изменять адреса проекта без необходимости вносить правки в десятки файлов.

## Часть 2: Жизненный цикл запроса

Давайте проследим путь запроса от момента, когда пользователь вводит URL в браузере, до отображения страницы.

**Пример URL:** `https://doc.m2profi.pro/sahmatka/ctrind.php?ctr=doc&act=index`

### 2.1. Схема работы фреймворка

```mermaid
graph TD
    A[Пользователь -> URL: /ctrind.php?ctr=doc&act=index] --> B{sahmatka/ctrind.php};
    B --> C[sahmatka/config.php];
    C --> D[Подключение к БД];
    C --> E[Загрузка /core/classes/*];
    C --> F[Создание объекта $r = new router()];
    B --> G[sahmatka/incudes_/header.php];
    G --> H{Проверка авторизации};
    H -- Да --> I[Выполнение $r->action_content()];
    H -- Нет --> J[Показ формы входа & exit()];
    I --> K{Роутер анализирует 'ctr' и 'act'};
    K --> L[Находит и подключает saḥmatka/fw/controllers/ctr__doc.php];
    L --> M[Создает объект new ctr__doc()];
    M --> N[Вызывает метод act__index()];
    N --> O{Контроллер получает данные из БД};
    O --> P[Контроллер вызывает $this->tpl(..., 'doc', 'template_name')];
    P --> Q[Загрузка saḥmatka/fw/templates/doc/template_name.php];
    Q --> R[Генерация HTML контента];
    R --> S[Возврат HTML в ctrind.php];
    B --> T[sahmatka/incudes_/foother.php];
    T --> U[Отправка полной страницы пользователю];
```

### 2.2. Файлы, участвующие в запросе

Для обработки одного стандартного запроса фреймворк задействует целый ряд файлов. Понимание роли каждого из них — ключ к пониманию всей системы.

1.  **`sahmatka/ctrind.php` (Точка входа)**
    *   **Назначение:** Главный файл, который принимает HTTP-запрос. Он не содержит бизнес-логики, а лишь организует процесс, подключая другие компоненты в правильном порядке (конфиг, шапку, роутер, подвал).
    *   **Особенность:** Является "каркасом" для стандартных HTML-страниц.

2.  **`sahmatka/config.php` (Конфигурация)**
    *   **Назначение:** Настройка всего приложения. Загружается самым первым.
    *   **Обязанности:**
        *   Управление отображением ошибок.
        *   Подключение к базе данных.
        *   Подключение всех файлов из `core/classes/`.
        *   Создание глобального экземпляра роутера (`$r`).

3.  **`sahmatka/incudes_/header.php` (Шапка и Аутентификация)**
    *   **Назначение:** Двойная роль. Во-первых, он выводит верхнюю часть HTML-страницы (доктайп, head, меню). Во-вторых, и это главное, он работает как "страж", проверяя сессию пользователя.
    *   **Особенность:** Если пользователь не авторизован, этот файл прерывает выполнение всего последующего кода (`exit()`), не давая дойти до логики контроллера.

4.  **`core/classes/router.php` (Класс Маршрутизатора)**
    *   **Назначение:** Определяет, какой контроллер и какой метод должны обработать текущий запрос на основе URL.
    *   **Особенность:** Не выполняет логику сам, а делегирует ее контроллеру.

5.  **`core/classes/controller.php` (Базовый класс Контроллера)**
    *   **Назначение:** Предоставляет общую функциональность (работа с БД, шаблонами) для всех дочерних контроллеров. Сам по себе напрямую в запросе не используется, но его код выполняется как часть дочернего контроллера.
    *   **Особенность:** Все контроллеры должны его наследовать.

6.  **`sahmatka/fw/controllers/ctr__{имя}.php` (Конкретный контроллер, например, `ctr__doc.php`)**
    *   **Назначение:** Содержит основную бизнес-логику страницы. Получает данные, обрабатывает ввод пользователя и решает, что показать.
    *   **Особенность:** Файл подключается роутером "на лету" (`include_once`).

7.  **`sahmatka/fw/templates/{имя}/{шаблон}.php` (Шаблон, например, `doc/index_template.php`)**
    *   **Назначение:** Отвечает за отображение данных в HTML. Содержит разметку и минимальное количество PHP для вывода переменных.
    *   **Особенность:** Подключается из метода `tpl()` базового контроллера.

8.  **`sahmatka/incudes_/foother.php` (Подвал)**
    *   **Назначение:** Выводит нижнюю часть HTML-страницы (закрывающие теги, подключение JS-скриптов).
    *   **Особенность:** Гарантирует, что страница всегда будет иметь завершенную структуру.

## Часть 3: Роутер (Маршрутизатор)

Роутер — это класс, который решает, какой код должен быть выполнен в ответ на запрос.

**Файл:** `core/classes/router.php`
**Класс:** `router`

### 3.1. Глубокий анализ работы роутера

Давайте разберем работу ключевого метода `action_content()` по шагам, чтобы понять его внутреннюю логику.

**Входные данные:** `$_GET['ctr'] = 'doc'`, `$_GET['act'] = 'index'`

1.  **Определение контроллера и действия:**
    *   Метод проверяет, были ли ему переданы аргументы `$controller` и `$action`. Если нет, он берет их из глобального массива `$_GET`, используя ключи, заданные в свойствах класса (`$this->ctr_get` и `$this->act_get`).
    *   Если `$_GET` пуст, по умолчанию используются значения `'index'`.
    *   *Результат:* `$controller = 'doc'`, `$action = 'index'`.

2.  **Формирование имен:**
    *   Роутер добавляет стандартные префиксы для формирования имени класса и имени метода.
    *   Имя класса: `'ctr__' . 'doc'` -> `'ctr__doc'`.
    *   Имя метода: `'act__' . 'index'` -> `'act__index'`.

3.  **Поиск и подключение файла контроллера:**
    *   Роутер формирует путь к файлу: `$this->ctr_dir . $class . '.php'` -> `'fw/controllers/' . 'ctr__doc' . '.php'`.
    *   Проверяется существование этого файла с помощью `file_exists()`.
    *   Если файл найден, он подключается с помощью `include_once`. Это гарантирует, что код класса будет загружен только один раз за время выполнения скрипта.
    *   Если файл не найден, метод записывает ошибку в лог и возвращает `false`, прекращая обработку.

4.  **Проверка существования класса:**
    *   После подключения файла роутер выполняет проверку `class_exists($class)`. Это важно, так как файл может существовать, но не содержать нужного определения класса (например, из-за опечатки в имени класса внутри файла).
    *   Если класс не найден, работа также прекращается.

5.  **Создание объекта и вызов метода:**
    *   Роутер создает экземпляр класса контроллера: `$obj = new ctr__doc();`.
    *   Затем он вызывает у этого объекта нужный метод: `$obj->act__index();`.
    *   На этом этапе управление полностью переходит к контроллеру, который выполняет свою бизнес-логику.

6.  **Возврат результата:**
    *   После успешного вызова метода `action_content()` возвращает `true`.

Этот пошаговый процесс обеспечивает предсказуемую и надежную загрузку и выполнение кода, отвечающего за конкретную страницу.

### 3.2. Вспомогательные методы

-   `get_object($controller)`:
    -   Вспомогательный метод, который инкапсулирует логику поиска файла контроллера и создания его объекта.
-   `acturl($controller, $action, ...)` и `actlink(...)`:
    -   Удобные функции для генерации URL-адресов и HTML-ссылок внутри системы. Они автоматически подставляют правильные GET-параметры `ctr` и `act`.

## Часть 4: Контроллеры

Контроллер — это класс, который содержит логику обработки запроса. Он связывает модели (данные) и представления (отображение).

### 4.1. Базовый контроллер `ctr__`

Все контроллеры в проекте **должны** наследоваться от базового класса `ctr__`. Это дает им доступ к мощному набору инструментов.

**Файл:** `core/classes/controller.php`
**Класс:** `ctr__`

#### Основные возможности:

-   **Доступ к БД**: В конструкторе каждый контроллер автоматически получает доступ к объекту для работы с базой данных через свойство `$this->mysql`.

-   **Шаблонизатор (метод `tpl()`)**:
    -   `$this->tpl($data, $ctr, $tpl_name)`
    -   Это правильный способ отображения HTML.
    -   **Как это работает:**
        1.  Вы вызываете этот метод в своем контроллере.
        2.  Он находит файл шаблона по пути `fw/templates/{$ctr}/{$tpl_name}.php`.
        3.  Массив `$data`, который вы передали, становится доступен внутри шаблона как переменная `$data`.
        4.  Это позволяет полностью отделить PHP-логику от HTML-разметки, делая код чище.

-   **CRUD-хелперы**:
    -   Класс предоставляет готовые методы для быстрого создания таблиц с данными (`display_table`, `display_ajax_crud`), форм и панелей управления. Это значительно ускоряет разработку стандартных административных интерфейсов.

-   **Методы для работы с данными**:
    -   `getid($id)`: Получает одну запись из таблицы по ее ID.
    -   `getfiltr($filtr)`: Получает массив записей по заданным условиям.
    -   **Важно:** Эти методы ожидают, что в вашем дочернем контроллере вы определите свойство `$this->table` (имя таблицы), `$this->key_filed` (ключевое поле) и метод `get_base_sql()`, который будет возвращать основной SQL-запрос для выборки данных.

### 4.2. Создание собственного контроллера

Рассмотрим процесс создания на примере `ctr__doc`.

**Файл:** `sahmatka/fw/controllers/ctr__doc.php`

```php
<?php
// 1. Имя класса должно начинаться с "ctr__"
// 2. Класс должен наследоваться от "ctr__"
class ctr__doc extends ctr__
{
    // Свойства, специфичные для этого контроллера
    var $table = 'fw_nodes';
    var $key_filed = 'fw_node_id';
    var $ctr = 'doc'; // Короткое имя для генерации URL

    // Конструктор (если нужен)
    function __construct()
    {
        // Вызов родительского конструктора не обязателен,
        // но может быть полезен.
        // parent::__construct();
    }

    // 3. Методы-действия должны начинаться с "act__"
    function act__index()
    {
        global $t; // Глобальная переменная для заголовков и мета-тегов
        $t['h1'] = 'Документы';

        // Получаем данные (например, из БД)
        $data = $this->getfiltr(['del' => 0]);

        // 4. Отображаем данные через шаблон (рекомендуемый способ)
        $this->tpl($data, 'doc', 'index_template');

        /*
        // Старый способ (не рекомендуется):
        // Смешение логики и HTML
        echo "<h1>Документы</h1>";
        foreach($data as $item) {
            echo "<div>{$item['title']}</div>";
        }
        */
    }

    function act__edit()
    {
        // Логика для отображения и обработки формы редактирования
        $id = (int)$_GET['id'];
        $item = [];
        if ($id > 0) {
            $item = $this->getid($id);
        }

        if ($_POST) {
            // Логика сохранения данных
            // ...
        }

        // Передаем данные в шаблон формы
        $this->tpl($item, 'doc', 'edit_form_template');
    }
}
```

### Резюме по контроллерам:

1.  **Имя файла:** `ctr__{name}.php`.
2.  **Имя класса:** `ctr__{name}`.
3.  **Наследование:** `extends ctr__`.
4.  **Имя метода-действия:** `act__{action}`.
5.  **Отображение:** Используйте `$this->tpl()` для разделения логики и представления.
